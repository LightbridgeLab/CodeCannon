## Conventions for doc files

- **One topic per file.** Keep each doc self-contained and focused.
- **Start with a purpose line.** First line after the title should say what the doc covers and when it's useful.
- **Be concise and actionable.** Write for an agent that needs to act, not a human reading for background. Prefer rules and examples over explanations.
- **Keep files under 200 lines.** If a topic grows beyond that, split it. Long docs defeat the purpose of selective loading.
- **Flat structure by default.** Only create subdirectories when you have 10+ docs and clear groupings.
